# Objectives for Module 3 – Authorisation

## Knowledge

The student will need knowledge of the following topics for Module 3:

* The distinction between authentication and authorisation, and why both are required in a secure system.
* The core principles of access control: least privilege, deny by default, and segregation of duties.
* The difference between implicit and explicit authorisation, including the confused deputy problem and its mitigations.
* The main access control models: access control list (ACL), role-based access control (RBAC), attribute-based access control (ABAC), and when each is appropriate.
* Object capabilities (OCAP) as an alternative to traditional authorisation schemes: capabilities as unforgeable tokens of authority, the principle of least authority (POLA), and capability-inspired patterns in modern web systems.
* Session management from an authorisation perspective: session timeout, session renewal, and their role in maintaining a trustworthy authorisation state.
* Session fixation and session hijacking: how these attacks work and how to defend against them.
* Secure logout and re-authentication practices.
* The differences between self-contained tokens and cookie-based sessions, including their respective security trade-offs.
* The security of self-contained tokens, with particular focus on JSON Web Tokens (JWTs): structure, signing algorithms, common vulnerabilities, and best practices.
* Object- and property-level authorisation (OLA) and the Insecure Direct Object Reference (IDOR) vulnerability class.
* Function-level access control (FLAC) and the risks of missing server-side enforcement.
* Common authorisation mistakes made by developers, including reliance on client-side controls.
* The role of auditing and monitoring in detecting and responding to authorisation failures.
* Best practices for testing access control, including both manual and automated approaches.

## Skills

The student will be able to demonstrate the following skills:

* Apply RBAC, ABAC, and capability-inspired patterns when designing the authorisation layer of a web application.
* Implement object- and property-level authorisation checks that correctly validate the relationship between the authenticated user and the requested resource or field.
* Implement function-level access control checks on server-side endpoints, without relying on UI-level restrictions.
* Evaluate a self-contained token implementation for common security weaknesses, including algorithm confusion, missing claims validation, and insecure storage.
* Design and execute access control tests that cover multiple privilege levels, positive and negative cases, object-level and property-level assertions.
* Identify common authorisation mistakes in code review and propose appropriate remediations.

## Competencies

With the knowledge and skills above, a developer is able to design and implement a complete authorisation layer for a modern web application.
This includes selecting an appropriate access control model, enforcing privilege boundaries at the object and property level, securing session and token artefacts, and verifying the implementation through structured testing.
The developer understands how authorisation failures manifest as real-world vulnerabilities – including broken access control, IDOR, confused deputy, and privilege escalation – and can apply the principle of least privilege consistently across an application’s design and codebase.
